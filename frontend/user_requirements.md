# User Requirements Document (URD)
**Project Name:** Vaaksetu Medical AI Platform (Frontend)

## 1. Overview
The frontend application serves as the primary user interface for the Vaaksetu Medical AI Platform. It allows users (patients/medical professionals) to interact with the underlying AI technologies, including speech recognition (ASR), symptom logic (LLM), and text-to-speech (TTS).

## 2. User Interfaces & Navigation
* **Dashboard/Home Screen:** A welcoming, medically-themed interface providing quick access to new sessions, past history, and voice interactions.
* **Responsive Design:** The interface must adapt seamlessly for mobile, tablet, and desktop environments.
* **Accessibility:** Must adhere to web accessibility standards (WCAG) so users with impairments can navigate effectively using screen readers and keyboard navigation.
* **Modern Aesthetics:** Clean, professional UI utilizing Tailwind CSS, ensuring clarity of medical information and an intuitive user experience.

## 3. Functional Requirements

### 3.1 Audio & Voice Interaction
* **Microphone Input:** Users can click a "Record" button or standard microphone icon to dictate their symptoms or queries via voice.
* **Audio Capture & Streaming:** The frontend must capture audio and securely transmit the payload to the backend for Automatic Speech Recognition (ASR).
* **Audio Playback:** The frontend must support playing audio responses generated by the Text-to-Speech (TTS) module from the backend.

### 3.2 Medical Chat & Data Input
* **Text Fallback/Input:** A standard text input field for users who prefer typing rather than speaking.
* **Real-time Feedback:** Visual indicators displaying system thinking, loading states, and audio recording progress.
* **Response Rendering:** Display responses from the LLM, properly formatting medical terms, follow-up questions, lists, and diagnostic insights.

### 3.3 Session Management
* **Conversation History:** Ability to view past queries or medical chat sessions.
* **Export/Download:** Option for users to download their session summary or LLM diagnosis.

## 4. Technical / Non-Functional Requirements
* **Framework:** React + Vite.
* **Styling:** Vanilla CSS / Tailwind CSS for responsive and modern interface development.
* **Performance:** Minimal page load times; smooth transitions and micro-animations to enhance user engagement.
* **API Communication:** Securely communicate with the Python Flask backend via RESTful APIs or WebSockets for real-time interactions.

## 5. Security & Privacy
* **Data Privacy:** Do not store sensitive voice or medical data on the client side locally without user-explicit consent or secure encryption.
* **Authentication (if applicable):** Secure login and protected routes for patient data.
